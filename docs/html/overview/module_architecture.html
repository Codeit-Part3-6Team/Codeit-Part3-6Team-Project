RAG 모듈 구조

새 기능을 붙일 때는 먼저 RAG 경로를 봅니다. 기존 학습/분류 코드는 참고용으로 남아 있습니다.

실행 흐름

configs/experiments/rag
->
scripts/run_rag_*
->
src/rag
->
experiments
->
reports

RAG 내부 모듈

모듈책임
document_loader.py원본 문서 로딩
engines/LangChain 기본 엔진과 local fallback 엔진
chunker.pylocal fallback용 chunk 생성
embedder.pylocal fallback용 embedding 생성
vector_store.pylocal fallback용 embedding 검색
retriever.pylocal keyword, semantic, hybrid retrieval
answerer.py답변과 citation 생성
adapters.pylocal fallback용 구현체 선택
pipeline.pyingest, retrieve, chat, evaluate 연결
validation.pyRAG config와 입력 경로 점검

공통 인프라

모듈책임
src/config.pyYAML config 로딩
src/artifacts.py산출물, 상태 파일, 실패 로그, 백업
src/experiments.py실험 summary 생성
src/utilspath, seed, logging 유틸

참고용 기존 예제

src/train.py, src/predict.py, src/models/는 기존 분류/HuggingFace 학습 파이프라인 참고용입니다. RAG 기능 추가의 기본 위치는 아닙니다.
