RAG 파이프라인 설명

이 프로젝트는 모델을 epoch 단위로 학습시키는 프로젝트가 아니라, 문서를 근거로 질문에 답하는 RAG 파이프라인을 만드는 프로젝트입니다.

한 줄 흐름

문서 준비 -> chunk 분할 -> embedding/index -> 검색 -> 답변 -> citation/평가

단계별 의미

1. 문서 로딩

PDF, HWP, DOCX, TXT 같은 원본 문서에서 텍스트와 metadata를 추출합니다.

2. Chunking

긴 문서를 검색하기 좋은 작은 근거 단위로 나눕니다.

3. Embedding

chunk와 질문을 비교할 수 있도록 vector로 바꿉니다.

4. Retrieval

질문과 관련 있는 top-k chunk를 찾습니다.

5. Answer

검색된 근거를 바탕으로 답변을 만들고 citation을 붙입니다.

6. Evaluation

정답 근거를 찾았는지, citation이 맞는지, 근거 없는 답변이 있는지 확인합니다.

일반 학습 파이프라인과 차이

일반 MLRAG
epoch를 돌며 model weight를 학습문서 chunk, 검색 방식, 답변 방식을 조정
best checkpoint가 핵심 산출물검색 근거, 답변, citation, 실패 사례가 핵심 산출물
accuracy, loss, f1 중심retrieval hit rate, citation correctness, unsupported answer 중심
prediction이 맞았는지 확인왜 그렇게 답했는지 근거까지 확인
