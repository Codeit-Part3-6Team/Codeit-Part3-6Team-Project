2. 멀티턴 대화 (Memory)

어떻게 동작하는가

사용자 질문→ 이전 대화 히스토리 조회→ 맥락 포함 검색→ 답변 생성→ 히스토리 저장

pipeline.py — run_rag_chat_with_history() (신규 52줄)

구성 요소설명
_CHAT_HISTORYthread_id를 키로 하는 대화 기록 저장소. 최대 10턴까지 유지
run_rag_chat_with_history()Config의 memory.enabled가 true일 때만 히스토리 활성화. 이전 3턴을 질문 앞에 컨텍스트로 추가하여 검색

Config — rag.answerer.memory (신규)

# configs/experiments/rag/rag_langchain.yaml
  answerer:
    mode: extractive
    provider: local
    # ...
    memory:
      enabled: false        # true로 바꾸면 멀티턴 대화 활성화

validation.py — _validate_memory_config() (신규 7줄)

memory.enabled 값이 bool인지 검증. reranker 경고문구 제거 (기존에 "not implemented"라고 되어있던 부분).

3. Re-rank (재정렬)

어떻게 동작하는가

1차 검색 (fetch_k = top_k × 3)→ Cross-Encoder로 정밀 재점수→ 상위 top_k개만 선택

engines/langchain.py — retrieve() 수정 + _rerank() 신규

변경설명
retrieve() fetch_k 확장reranker.enabled=true 면 1차 검색에서 top_k의 3배를 가져옴
_rerank() 신규BAAI/bge-reranker-v2-m3 Cross-Encoder로 질문-문서 쌍 재점수 후 상위 top_k만 반환
의존성sentence-transformers 패키지 필요. 없으면 경고 후 1차 결과 그대로 사용

Config — rag.reranker (기존 자리, model_name 채움)

# configs/experiments/rag/rag_langchain.yaml
  reranker:
    enabled: false              # true로 바꾸면 Re-rank 활성화
    provider: huggingface       # Cross-Encoder 사용
    model_name: BAAI/bge-reranker-v2-m3  # 기본 모델
    top_k: 3                    # 최종 반환 개수

requirements.txt

sentence-transformers>=3.0.0 추가. Cross-Encoder 재정렬 모델을 위한 의존성. 없으면 reranker가 경고를 내고 건너뜀.
