# Kickoff Markdown

팀원에게 프로젝트를 처음 설명할 때 사용할 Markdown 원본 문서를 둡니다.

## 문서 목록

| 문서 | 용도 |
| --- | --- |
| `KICKOFF_GUIDE.md` | 킥오프 설명 원본 |
| `TEAM_BRIEFING_FLOW.md` | 팀원에게 설명할 순서와 강조점 |

## 사용 기준

- 실제 발표/공유용 HTML은 `docs/html/kickoff/`에서 확인합니다.
- 설명 흐름을 바꾸면 `TEAM_BRIEFING_FLOW.md`를 먼저 갱신합니다.
- 역할, 일정, 운영 방식이 바뀌면 `docs/md/workflow/` 문서도 함께 확인합니다.
