DE 역할에 따라 RAG 파이프라인에 필요한 데이터를 준비하고 품질을 검증했습니다.
구체적으로는 원본 문서 상태 확인, 텍스트 품질 검증, 평가 질문 CSV 생성, 파일 정리 순서로 진행했습니다.

## 1. 데이터 현황 확인

### 진행 내용

`data/raw/data_list.csv`를 로드해 전체 문서 수와 기본 통계를 확인했습니다.

| 항목 | 내용 |
|---|---|
| 총 문서 수 | 100개 |
| 파일형식 | hwp 96개, pdf 4개 |
| 텍스트 평균 길이 | 3,843자 |
| 텍스트 최대 길이 | 18,335자 |
| 텍스트 최소 길이 | 89자 |

### 선택 이유

`data_list.csv`의 텍스트 컬럼에는 원본 hwp/pdf 파일에서 미리 추출한 본문이 들어있습니다.
RAG는 이 텍스트를 chunk로 나눠 검색에 사용하므로, 텍스트 품질이 전체 파이프라인 품질을 결정합니다.
따라서 원본 파일 하나씩 열어보기 전에 텍스트 컬럼 상태를 먼저 점검했습니다.

## 2. 텍스트 품질 검증 및 이슈 문서 제외

### 진행 내용

텍스트 길이가 **500자 미만**인 문서 7개를 이슈 문서로 분류하고, 원본 hwp 파일과 대조해 원인을 확인했습니다.

확인 결과 파일 자체 손상이 아닌 **텍스트 추출 실패**로 판단됩니다.
목차/표지/안내문만 추출되고 본문이 누락된 패턴이며, 원본 파일은 정상일 가능성이 높습니다.

이후 회의를 통해 데이터 문자를 직접 추출할지 말지 결정하는게 나아보임

| 공고 번호 | 사업명 | 파일형식 | 텍스트 길이 | 추출된 내용 | 이슈 |
|---|---|---|---|---|---|
| 20240827859 | EIP3.0 고압가스 안전관리 시스템 구축 용역 | hwp | 234자 | 목차까지만 추출, 본문 누락 | 텍스트 추출 실패 |

### 선택 이유

500자 기준을 선택한 이유는 RAG config의 기본 `chunk_size`가 500자이기 때문입니다.
문서 전체가 chunk 하나보다 짧으면 검색 자체가 의미 없고, 평가 질문을 만들 근거도 부족합니다.
이슈 문서는 삭제하지 않고 보존하되, 평가 질문 생성 대상에서만 제외했습니다.

## 3. 평가 질문 CSV 생성

### 진행 내용

93개 정상 문서를 대상으로 두 차례에 걸쳐 평가 질문을 생성했습니다.

#### 1차 생성 (317개) — 문제 발견으로 폐기

키워드 주변 텍스트를 그대로 정답으로 사용하는 방식으로 자동 생성했으나,
리뷰 과정에서 아래 구조적 문제를 발견했습니다.

| 카테고리 | 문제 |
|---|---|
| 일정 | "사업기간" 키워드 뒤 텍스트가 목차 페이지 번호인 경우 다수 |
| 자격 | "참가자격" 키워드 뒤가 항목명만 나오고 실제 요건 없음 |
| 제출서류 | "제출서류" 키워드 뒤가 목차 챕터명으로 채워진 경우 많음 |

카테고리별 품질 점수 (리뷰 기준):
- 예산: 9.5/10, 발주기관: 9/10, 일정: 5/10, 자격: 4/10, 제출서류: 3/10

#### 2차 생성 (897개) — 최종 채택

문제를 해결하기 위해 두 가지를 개선했습니다.

**개선 1: LLM 의미 기반 추출 도입**
일정/자격/제출서류는 키워드 추출 대신 OpenAI GPT-4o-mini로 문서 의미를 파악해 정답을 추출했습니다.
예: "사업기간 종합산정서 별표7..." → "계약일로부터 6개월"

**개선 2: 질문 다양성 추가**
실제 사용자는 같은 정보를 다양한 표현으로 질문합니다.
카테고리별로 질문 변형 3개를 생성해 Retriever 평가 신뢰도를 높였습니다.

### 최종 결과

- 출력 파일: `data/raw/eval_questions.csv`
- 생성 스크립트: `scripts/generate_eval_questions.py`
- 총 질문 수: **897개**

| 카테고리 | 질문 수 | 추출 방법 |
|---|---|---|
| 발주기관 | 279개 | 발주 기관 컬럼 직접 사용 × 3종 질문 |
| 예산 | 273개 | 사업 금액 컬럼 직접 사용 × 3종 질문 |
| 일정 | 213개 | LLM 의미 기반 추출 × 3종 질문 |
| 제출서류 | 114개 | LLM 의미 기반 추출 × 3종 질문 |
| 자격 | 18개 | LLM 의미 기반 추출 (정보 없는 문서 자동 제외) |

### 참고사항

- `expected_chunk_id`는 현재 비어있음
- VM에서 RAG ingest 파이프라인 실행 후 chunk ID 생성되면 채워야 함
- 자격 카테고리 18개는 LLM이 정보가 충분하다고 판단한 문서만 포함 (품질 우선)
