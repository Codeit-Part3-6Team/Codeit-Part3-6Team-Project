## 1. 진행 경과

### 1.1 Agent Loop 계획 평가

`fix/answer-quality` 완성 시점의 코드베이스를 기준으로 `configs/experiments/rag/rag_agent.yaml`(194줄)의 각 키와 `src/` 구현 코드를 1:1 대조 감사.

| 평가 항목 | 결과 |
|-----------|------|
| 원계획 7개 Config→코드 갭 | **모두 실재하는 갭으로 확인.** `agent.enabled` 미참조, `NotImplementedError` 등 |
| 원계획 3개 하드코딩 결합 | **정확.** `pipeline.py` Phase 강제호출, `langchain.py` 이원화 등 |
| 구현 순서 | **논리적.** 다만 `langchain.py` 통합을 7→6단계로 당길 것을 제안 (reranker 접근성) |

### 1.2 추가 감사: 계획에 없던 문제 22건

`src/rag/` 전체를 대상으로 하드코딩 값, config 키 혼선, dead config, 코드 중복, 모듈 간 결합을 전수 스캔.

| 구분 | 건수 | 대표 사례 |
|------|:---:|-----------|
| Config 키 혼선 | 3건 | `rag.chunk` vs `rag.splitter`, `max_tokens` 256 vs 512, `rag.llm` legacy fallback |
| Dead config | 5건 | `backup.*`, `metric.*`, `rag.answerer.require_citations`, `rag.chunk.unit`, `rag.embedding.normalize` |
| 코드 중복 | 5건 | `_dot()` 3곳, `_citations_from_chunks()` 2곳, `_resolve_path()` 5곳 등 |
| 확장성 저해 | 9건 | Reranker `max_length=512` 하드코딩, Judge OpenAI 전용, CSV 컬럼명 한국어 하드코딩, 키워드 점수 하드코딩 등 |

### 1.3 Config 인터페이스 평가

`rag_agent.yaml` + `configs/README.md` 기준.

| 발견 | 내용 |
|------|------|
| README 누락 | `agent.*` 섹션 전체가 문서화되지 않음. `evaluation.llm_judge`도 누락 |
| `output_schema` 정의 위치 없음 | `facts_schema`, `decision_schema` 참조만 있고 필드 정의는 config 어디에도 없음 |
| Tool별 `prompt_template` 없음 | Tool마다 다른 프롬프트 필요하나 오버라이드 경로 없음 |
| Phase 간 데이터 흐름 미정의 | `depends_on`은 순서만 정의. 어떤 Tool의 어떤 출력을 받을지 불명 |
| Tool 실패 정책 없음 | LLM 호출 실패 시 분기 정의 없음 |
| 병렬 실행 제어 없음 | Phase 내 여러 Tool의 병렬/직렬 여부 설정 불가 |

→ 계획 부록 B에 `agent.schemas`, `tools.*.prompt_template`, `tools.*.input_from`, `tools.*.on_failure`, `phases[*].parallel` 보강안 제시.

## 2. 의사결정 사항

| 결정 | 근거 |
|------|------|
| **Config만으로 L+까지 가능** | Phase DAG + Tool + output_schema + company_profile 주입으로 충분 |
| **XL은 Config + Tool plugin 2~3개 필요** | Rule 엔진, 외부DB, cross-doc citation chaining은 별도 구현 대상 |
| **langchain.py 통합 순서 당김** | 7단계→6단계. Reranker 등 고급 기능을 agent 경로에서 초기부터 사용 가능하게 |
| **`output_schema` 정의는 Config에** | schema 추가 시 코드 수정 없이 실험할 수 있도록 (코드 하드코딩 지양) |
| **데모 앱 연동은 Agent Loop 완성 후** | 구조적 문제 없음 확인. `mock_data.py` → 브릿지 함수 10줄로 교체 가능 |
